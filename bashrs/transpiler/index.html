Builder API for programmatic transpilation

§Transpiler Builder API

Provides a builder pattern for programmatic transpilation in build scripts, xtask commands, and CI/CD pipelines. This is the recommended API for integrating bashrs into your build process.

§Examples

§Basic Usage

use bashrs::Transpiler;

Transpiler::new()
    .input("src/install.rs")
    .output("target/install.sh")
    .transpile()?;

§Git Hooks in xtask

// xtask/src/main.rs
use bashrs::Transpiler;

fn transpile_hooks() -> bashrs::Result<()> {
    Transpiler::new()
        .input("hooks/pre-commit.rs")
        .output(".git/hooks/pre-commit")
        .permissions(0o755)
        .transpile()?;

    Ok(())
}

§build.rs Integration

// build.rs
use bashrs::Transpiler;

fn main() {
    println!("cargo:rerun-if-changed=src/install.rs");

    Transpiler::new()
        .input("src/install.rs")
        .output("target/install.sh")
        .permissions(0o755)
        .transpile()
        .expect("Failed to transpile install script");
}
