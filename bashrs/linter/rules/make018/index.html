MAKE018: Parallel-unsafe targets (race conditions)

Rule: Detect targets that modify shared state without synchronization

Why this matters: When Make runs with -j (parallel jobs), targets can run concurrently. Targets that write to the same file or directory can have race conditions. This can cause corrupted builds or intermittent failures.

Auto-fix: Suggest .NOTPARALLEL or order-only prerequisites

§Examples

❌ BAD (parallel-unsafe - multiple targets write to same directory):

install-bin:
\tcp app /usr/bin/app

install-lib:
\tcp lib.so /usr/bin/lib.so

✅ GOOD (with .NOTPARALLEL):

.NOTPARALLEL:

install-bin:
\tcp app /usr/bin/app

install-lib:
\tcp lib.so /usr/bin/lib.so

✅ GOOD (with explicit dependencies):

install-bin:
\tcp app /usr/bin/app

install-lib: install-bin
\tcp lib.so /usr/bin/lib.so
