MAKE008: Tab vs spaces in recipes (CRITICAL)

Rule: Detect spaces instead of tabs in recipe lines (fatal Make error)

Why this matters: GNU Make REQUIRES recipe lines to start with a TAB character (ASCII 0x09). Using spaces instead of tabs causes a fatal “missing separator” error. This is one of the most common and frustrating Make errors.

Auto-fix: Replace leading spaces with a single tab

§Examples

❌ BAD (spaces - will fail):

build:
    gcc main.c -o app

✅ GOOD (tab):

build:
    gcc main.c -o app
