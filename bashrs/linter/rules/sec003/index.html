SEC003: Command Injection via find -exec with sh -c

Rule: Detect {} embedded in shell command strings within find -exec sh -c

Why this matters: When {} appears inside a shell command string (sh -c ‚Äò‚Ä¶{}‚Ä¶‚Äô), filenames with special characters can lead to command injection. The {} is expanded by find BEFORE the shell parses the string, allowing malicious filenames to inject arbitrary commands.

Important: Unquoted {} as a separate argument (find -exec rm {} ;) is SAFE because find passes the filename as a single argument. The shell never interprets it.

Auto-fix: Use positional parameters instead
