1//! Quality Gates Module (ML-001 through ML-017)
2//!
3//! Implements tiered quality gate enforcement following Toyota Production System
4//! principles. Gates are configured via `.pmat-gates.toml` and enforced at three tiers:
5//!
6//! - **Tier 1 (ON-SAVE)**: Sub-second checks (clippy, complexity)
7//! - **Tier 2 (ON-COMMIT)**: 1-5 minute checks (tests, coverage, SATD)
8//! - **Tier 3 (NIGHTLY)**: Hours (mutation testing, security audit)
9//!
10//! # Features
11//!
12//! - **Quality Gates** (ML-001 to ML-003): Tiered enforcement with configurable thresholds
13//! - **SBFL Fault Localization** (ML-004 to ML-006): Tarantula, Ochiai formulas
14//! - **Oracle ML Classification** (ML-007 to ML-010): k-NN classifier, pattern library, drift detection
15//! - **Rich ASCII Reporting** (ML-011 to ML-014): Box drawing, sparklines, histograms
16//! - **Control Flow Analysis** (ML-015 to ML-017): CFG generation, complexity metrics
17//!
18//! # Toyota Way Principles
19//!
20//! - **Jidoka** (Automation with human touch): ML classifies, human approves
21//! - **Kaizen** (Continuous improvement): Learn from fix acceptance
22//! - **Visual Management** (Mieruka): Rich ASCII dashboards
23//! - **Genchi Genbutsu** (Go and see): SBFL locates actual faults
24//! - **Poka-yoke** (Error-proofing): Confidence scores prevent bad fixes
25//!
26//! # References
27//!
28//! - BASHRS-SPEC-ML-001: ML-Powered Linting specification
29//! - Jones & Harrold (2005): Tarantula fault localization
30//! - Abreu et al. (2007): Ochiai formula
31//! - McCabe (1976): Cyclomatic complexity
32//! - Kim et al. (2013): Bug localization with learning-to-rank
