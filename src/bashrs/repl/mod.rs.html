1//! # bashrs REPL - Interactive Shell Analysis Environment
2//!
3//! The REPL (Read-Eval-Print Loop) module provides an interactive environment for
4//! analyzing, transforming, and debugging bash scripts in real-time.
5//!
6//! ## Overview
7//!
8//! The bashrs REPL is a production-ready interactive shell that combines:
9//!
10//! - **5 Interactive Modes**: Normal, purify, lint, debug, and explain modes
11//! - **Real-time Analysis**: Parse, lint, and purify bash code interactively
12//! - **Script Management**: Load scripts, extract functions, reload after changes
13//! - **Session State**: Persistent variables, history, and context
14//! - **Tab Completion**: Intelligent completion for commands, modes, and files
15//! - **Multi-line Input**: Natural support for functions, loops, and conditionals
16//!
17//! ## Architecture
18//!
19//! The REPL follows the **Debugger-as-REPL** pattern (matklad pattern):
20//! - Debugger capabilities are exposed through REPL commands
21//! - Symbiotic embedding enables deep integration with bash analysis tools
22//!
23//! ## Quick Start
24//!
25//! ```bash
26//! $ bashrs repl
27//! bashrs [normal]> echo "Hello, REPL!"
28//! Hello, REPL!
29//!
30//! bashrs [normal]> :parse echo test
31//! ‚úì Parse successful!
32//!
33//! bashrs [normal]> :mode lint
34//! Switched to lint mode
35//!
36//! bashrs [lint]> cat $FILE | grep pattern
37//! Found 1 issue(s): SC2086 - Unquoted variable
38//! ```
39//!

61//! ## Features
62//!
63//! ### Interactive Modes
64//!
65//! The REPL supports 5 specialized modes:
66//!
67//! | Mode | Purpose | Use Case |
68//! |------|---------|----------|
69//! | **normal** | Direct execution | Testing commands, learning bash |
70//! | **purify** | Automatic purification | Fixing idempotency issues |
71//! | **lint** | Automatic linting | Finding security problems |
72//! | **debug** | Step-by-step execution | Understanding complex scripts |
73//! | **explain** | Interactive explanations | Learning bash constructs |
74//!
75//! ### Commands
76//!
77//! Core commands for bash analysis:
78//!
79//! - `:parse <code>` - Parse bash code and show AST
80//! - `:lint <code>` - Lint for security and quality issues
81//! - `:purify <code>` - Transform to idempotent/deterministic code
82//! - `:mode <name>` - Switch to a different mode
83//! - `:load <file>` - Load and analyze a bash script
84//! - `:reload` - Reload the most recently loaded script
85//! - `:vars` - Show session variables
86//! - `:history` - Show command history
87//!
88//! ### Script Loading
89//!
90//! Load complete scripts for analysis:
91//!
92//! ```bash
93//! bashrs [normal]> :load deploy.sh
94//! ‚úì Loaded: deploy.sh (5 functions, 120 lines)
95//!
96//! bashrs [normal]> :functions
97//! Available functions (5 total):
98//!   1 validate_env
99//!   2 build_app
100//!   3 run_tests
101//!   4 deploy_staging
102//!   5 deploy_production
103//! ```
104//!
105//! ### Variable Management
106//!
107//! Persistent variables across your session:
108//!
109//! ```bash
110//! bashrs [normal]> app_name="myapp"
111//! ‚úì Variable set: app_name = myapp
112//!
113//! bashrs [normal]> version=1.0.0
114//! ‚úì Variable set: version = 1.0.0
115//!
116//! bashrs [normal]> echo $app_name v$version
117//! myapp v1.0.0
118//! ```
